### 任务复杂度判断

<!--
✅ 必填：定义什么情况是 simple/medium/complex
LLM 根据这些规则输出 complexity 字段
-->

判断用户任务属于哪个复杂度等级：

**SIMPLE（简单）**：
- 直接回答问题，不需要工具调用
- 单个工具调用即可完成
- 信息查询类（天气、时间、简单搜索）
- 闲聊、问候、确认

**MEDIUM（中等）**：
- 需要 2-5 个步骤完成
- 需要规划但步骤清晰
- 文件处理、数据分析、内容生成
- 涉及多个工具协作但逻辑简单

**COMPLEX（复杂）**：
- 需要 5 个以上步骤
- 涉及系统设计、架构决策
- 需要多轮验证和质量保证
- 跨领域综合任务


### 多智能体触发条件

<!--
✅ 建议填写：定义什么情况需要多智能体协作
LLM 根据这些规则输出 needs_multi_agent 字段
-->

以下情况需要多智能体协作（needs_multi_agent = true）：
- 任务涉及多个**独立领域**且需要专业知识
- 任务步骤之间**可并行执行**且耗时较长
- 需要多个角色**分工协作**
- 预估总执行时间超过 3 分钟

以下情况**不需要**多智能体：
- 任务步骤是串行依赖的
- 单个智能体可以顺序完成
- 任务简单，拆分反而增加开销


### 规划触发条件

<!--
✅ 建议填写：定义什么情况需要创建执行计划
LLM 根据这些规则输出 needs_plan 字段
-->

以下情况需要创建计划（needs_plan = true）：
- 任务需要 3 个以上步骤
- 步骤之间有依赖关系需要明确
- 可能需要根据中间结果调整后续步骤

以下情况**不需要**计划：
- 直接回答即可
- 单个工具调用
- 步骤简单且确定


### 持久化需求判断

<!--
⬜ 可选：定义什么信息需要跨会话记忆
LLM 根据这些规则输出 needs_persistence 字段
-->

以下信息需要持久化（needs_persistence = true）：
- 用户偏好设置
- 重要的任务结果
- 需要长期记住的信息
