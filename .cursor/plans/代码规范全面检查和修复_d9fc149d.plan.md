# 代码规范全面检查和修复计划

## 检查维度

根据 `.cursor/rules/01-general-conventions/RULE.mdc` 和其他规范文件，我已完成初步扫描，发现以下主要问题：

## 问题汇总

### 1. 类型注解缺失问题 (优先级：高)

**发现：**

- `core/` 目录下有 **210 个函数**缺少返回类型注解（`def func():`）
- 有 **19 个异步函数**缺少返回类型注解（`async def func():`）
- 根据规范要求："所有函数必须包含类型注解"

**影响范围：**

- [`core/confirmation_manager.py`](core/confirmation_manager.py) - 5 处
- [`core/billing/tracker.py`](core/billing/tracker.py) - 2 处
- [`core/context/context_engineering.py`](core/context/context_engineering.py) - 8 处
- [`core/context/prompt_manager.py`](core/context/prompt_manager.py) - 6 处
- [`core/tool/executor.py`](core/tool/executor.py) - 5 处
- [`core/llm/claude.py`](core/llm/claude.py) - 19 处
- [`core/agent/simple_agent.py`](core/agent/simple_agent.py) - 6 处
- 其他 65+ 个文件

**示例问题：**

```python
# 当前（不符合规范）
def set_response(self, response: str, metadata: Optional[Dict[str, Any]] = None):
    """设置用户响应"""
    self.response = response

# 应该（符合规范）
def set_response(self, response: str, metadata: Optional[Dict[str, Any]] = None) -> None:
    """设置用户响应"""
    self.response = response
```

- 这些空方法违反了"代码应该简洁有意义"的原则

**影响范围：**

- [`core/routing/complexity_scorer.py`](core/routing/complexity_scorer.py:90)
- [`core/confirmation_manager.py`](core/confirmation_manager.py:149)
- [`core/billing/tracker.py`](core/billing/tracker.py:51)
- [`core/monitoring/token_budget.py`](core/monitoring/token_budget.py:89)
- [`core/context/providers/knowledge.py`](core/context/providers/knowledge.py:27)
- [`core/context/providers/memory.py`](core/context/providers/memory.py:28)
- 其他 10+ 个文件

**示例问题：**

```python
# 当前（不必要的空方法）
def __init__(self):

# 应该（直接删除空 __init__）
# 不需要定义 __init__ 方法
```

### 5. 文档字符串问题 (优先级：中)

**发现：**

- 部分函数缺少完整的 Google 风格 docstring
- 特别是 `__init__` 方法和内部方法（`_method`）

**规范要求：**

```python
def function(arg1: str, arg2: int) -> bool:
    """
    函数简短描述
    
    Args:
        arg1: 参数1的说明
        arg2: 参数2的说明
        
    Returns:
        返回值说明
        
    Raises:
        ValueError: 什么情况下抛出
    """
    pass
```

### 6. 循环依赖和 TYPE_CHECKING 使用 (优先级：低)

**发现：**

- 有 **22 个文件**使用 `TYPE_CHECKING` 来避免循环依赖
- 这是正确的做法，但说明模块间耦合度较高
- 只有 **3 个文件**使用 `from __future__ import annotations`（Python 3.10+ 推荐方式）

**影响文件：**

- [`core/context/prompt_manager.py`](core/context/prompt_manager.py)
- [`core/agent/factory.py`](core/agent/factory.py)
- [`utils/background_tasks/service.py`](utils/background_tasks/service.py)
- 其他 19 个文件

## 建议修复步骤

由于问题较多，建议分阶段进行：

**阶段 1：快速提升（1-2小时）**

- 删除所有空 `__init__` 方法
- 修复高频使用的核心类的类型注解（如 `SimpleAgent`, `ToolExecutor`）

**阶段 2：系统性修复（3-5小时）**

- 批量补充函数返回类型注解
- 使用脚本辅助检测和修复

**阶段 3：架构优化（需要更长时间）**

- 拆分超长文件
- 优化模块依赖关系
