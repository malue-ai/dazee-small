## 二、智能体优化建议（按目标模块归类）

以下建议来自报告中的 `optimization_suggestions` 与 `pipeline_diagnosis`，已按 **framework / memory / prompt / skill / model** 归类并去重合并。

---

### 2.1 Framework（框架层）

| # | 问题 | 建议 |
|---|------|------|
| 1 | **规划死循环**：多次连续 plan 不进入执行（如 A3 中 4/5 调用为 plan） | 为 plan 调用设硬性上限（如连续 plan 不超过 2 次必须进入执行）；在 plan 模块中做收敛检测——若两次 plan 内容高度相似，强制进入执行阶段。 |
| 2 | **Token 灾难性膨胀**（A3 121K、C3 197K、B4 48K、B7 74K、B6 55K） | ① 每次 re-plan 对旧 plan 做 compaction（只保留最新 plan + 旧 plan 摘要）；② 设置 context 硬上限告警（如 >30K 触发强制裁剪）；③ 工具输出超过阈值（如 2K tokens）必须经 compaction 再写入上下文；④ 设置 `max_tool_result_tokens` 硬限制。 |
| 3 | **工具输出未 compaction**（C3：arXiv 等原始数据直接灌入） | 在 tool_result 写入对话上下文前必须经 compaction：超过阈值则摘要提取，只保留关键结构化信息；对论文类只保留 title/authors/abstract 等，过滤大体积字段。 |
| 4 | **任务未完成却无交付/反馈**（A3 回复为空、C3「马上就好」但已终止） | 实现 graceful degradation：在达到工具调用上限或 token 预算耗尽时，必须输出「已获取的部分结果 + 失败原因 + 后续建议」，禁止静默失败或虚假进度话术。 |
| 5 | **缺少交付物检查**（C3：7 次工具调用零有效产出） | 在工具调用链末尾增加 deliverable check：检查是否已生成用户要求的交付物；若未生成则触发兜底（用已有数据生成输出或明确说明失败原因）。 |
| 6 | **Plan 创建后不自动推进**（B4：创建 Plan 后仅追问、0 次文件操作） | Plan 工具后处理应能自动触发第一步执行（如第一步为「扫描目标路径」）；若路径未知则先扫描常见位置，避免在追问环节卡死。 |
| 7 | **长任务确认机制未在正确时机触发**（B4） | long_running_confirm 应在执行过程中（如超过 20 轮）触发，而非在规划阶段就阻塞；批量操作类任务应引导先预估工作量并自动推进。 |
| 8 | **双重 plan + HITL 打断**（C3） | 第一次 plan 前做可执行性预检（所需工具可用、数据源可达）；HITL 应有明确触发条件，避免在可自动化完成的任务中不必要中断。 |

### 2.2 Memory（记忆层）

| # | 问题 | 建议 |
|---|------|------|
| 1 | **显式自我介绍未写入**（B6：「我是产品经理」等未进入 user_profile） | 对显式自我陈述（「我是…」「我用…」「我喜欢…」）设最高优先级记忆写入规则，触发立即写入 user_profile，而非依赖隐式推断。 |
| 2 | **记忆检索污染/隔离不足**（B6：返回其他会话/用户内容） | ① Memory 按 user_id + session_id 强隔离；② 检索 user_profile 时做精确匹配或强约束的语义范围，避免跨会话模糊搜索；③ 为记忆增加来源 metadata（写入会话/时间）。 |
| 3 | **用户画像加载过大**（B6：55K input） | 用户画像以压缩结构化摘要存储（如 JSON：role, tools, preferences），加载时只注入该摘要（<500 tokens），不注入原始对话历史；Memory 检索结果需 relevance scoring + top-k 后再注入。 |
| 4 | **偏好冲突未检测与更新**（B7：简洁→详细 未覆盖） | 实现 Memory Conflict Resolution：同类偏好 last-write-wins，并保留变更历史；召回时同一维度多条记忆去重，返回最新版本；用 category+dimension（如 writing_style.verbosity）做偏好索引。 |
| 5 | **偏好类表达未触发 memory_store**（B7） | 在用户表达偏好类语句时自动触发 memory_store；增加偏好检测的 intent 或触发器，确保「我喜欢…」「我更喜欢…」不被当作闲聊忽略。 |
| 6 | **隐式信号未强化画像**（B6：写 PRD 未强化「产品经理」） | 当用户任务类型与已有画像高度一致时，强化该画像权重；不一致时触发画像更新或确认。 |

### 2.3 Prompt（提示词 / 系统指令）

| # | 问题 | 建议 |
|---|------|------|
| 1 | **不确定时反复规划而非执行**（A3） | 在系统提示词中明确：「当你不确定时，优先尝试执行并从结果中获取信息，不要反复修改计划；执行失败后再通过 RVR-B 回溯调整。」 |
| 2 | **路径不明时直接阻塞追问**（B4） | 当用户说「这个文件夹」但路径不明时，优先 ls 当前工作目录或桌面/下载/文档等常见路径，列出候选让用户确认；策略为「先探测，探测失败再追问」。 |
| 3 | **禁止虚假进度话术**（C3） | 硬约束：「永远不要告诉用户你即将完成但实际未完成的内容。若任务遇阻，直接说明问题与下一步计划。禁止在已终止时使用『稍等/马上就好』等表述。」 |
| 4 | **记忆召回后未体现在回复**（B7） | 强指令：「当 memory_recall 返回用户偏好时，必须在回复中体现该偏好，且不要重复询问用户已表达过的偏好；存在冲突时使用最近一条。」 |
| 5 | **已知信息仍追问**（B7） | 生成追问列表前，先检查 memory 与对话历史是否已有答案；有则跳过或改为确认式提问（如「你之前提到喜欢详细风格，这次也按这个来吗？」）。 |
| 6 | **行动优先原则**（B4） | 强调桌面 AI 应以「行动优先」：在有合理默认值或可推断上下文时，优先尝试执行而非追问；可加入 bias toward action 类指令。 |
| 7 | **记忆不确定时勿高置信度错误**（B6） | 当记忆检索结果与当前提问上下文明显不吻合时，应诚实说明「记忆可能不准确」或主动询问，不要强行拼凑错误画像。 |

### 2.4 Skill（技能 / 工具）

| # | 问题 | 建议 |
|---|------|------|
| 1 | **PDF 扫描件无检测与 OCR 降级**（A3） | 在 PDF 解析类 skill 中内置扫描件检测（如页面是否为纯图）；检测到扫描件时：先尝试 OCR 提取，失败则标注该页为「不可提取区域」并继续处理其余页。 |
| 2 | **arXiv 等返回体过大**（C3） | arXiv 类 skill 内置结果裁剪：限制条数（如 top 10）、只返回 title/authors/year/abstract，过滤 full_text、references 等大字段；在 skill 层控制输出体积。 |
| 3 | **Plan 第一步设计**（B4） | Plan 的 step 1 可设计为「扫描目标路径」；若路径未知则扫描常见位置，使 Plan 创建即能执行，减少在追问环节卡住。 |

---

### 2.5 Model（模型调用与推理）

| # | 问题 | 建议 |
|---|------|------|
| 1 | **复杂任务未用扩展思考**（A3/C3：thinking_tokens=0） | 对 complexity=complex 的任务强制启用扩展思考（extended thinking），帮助规划与错误恢复时的深层推理，减少低质量反复 re-planning。 |
| 2 | **超长上下文下推理能力下降**（C3：197K 下 thinking=0） | 在上下文超过一定阈值（如 50K）时强制启用 thinking/chain-of-thought；或通过 compaction 确保上下文不膨胀到该程度。 |
| 3 | **过于谨慎、倾向追问**（B4） | 在系统提示词中强调「行动优先」，并视情况在模型侧或路由侧对「可推断场景」减少追问、增加执行倾向。 |

## 三、建议优先级（按出现频率与影响面）

| 优先级 | 方向 | 代表动作 |
|--------|------|----------|
| **P0** | 上下文与 Token | 工具输出 compaction、context 硬上限与告警、plan 后旧内容压缩 |
| **P0** | 规划收敛 | plan 连续调用上限、收敛检测、强制进入执行 |
| **P0** | 记忆隔离与写入 | user_id/session 隔离、显式身份/偏好立即写入、冲突策略 last-write-wins |
| **P1** | 交付与降级 | deliverable check、graceful degradation、禁止虚假进度话术 |
| **P1** | 提示词 | 行动优先、先探测再追问、记忆召回必须体现在回复 |
| **P2** | Skill | PDF 扫描件+OCR 降级、arXiv 等结果裁剪、Plan 第一步可执行化 |
| **P2** | Model | 复杂任务启用扩展思考、超长上下文时强制 thinking 或限制上下文 |
