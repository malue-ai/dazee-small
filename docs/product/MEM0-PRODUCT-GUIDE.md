## 一、什么是 Mem0？

### 一句话定义

**Mem0 是 Dazee 智能助理的"长期记忆系统"**，让 AI 能够记住用户的偏好、习惯、工作内容，实现真正的"个性化服务"。

### 核心价值

| 传统 AI 对话 | Dazee + Mem0 |
|-------------|--------------|
| 每次对话从零开始 | 跨会话记忆，越用越懂你 |
| 通用回答 | 基于你的习惯个性化回答 |
| 用户重复说明背景 | AI 主动关联历史信息 |
| 无法主动提醒 | 智能提醒待办、截止日期 |

## 二、核心功能

### 2.1 自动记忆提取

**用户无需手动告知偏好**，Dazee 从日常对话中自动提取关键信息：

| 提取类型 | 示例对话 | 提取结果 |
|---------|---------|---------|
| **人物关系** | "下午要见采购部的老张" | 老张 → 永辉采购部 → 关键决策人 |
| **数字金额** | "合同金额 150 万" | 永辉合同 = 150 万 |
| **工作习惯** | "每周三要写周报" | 周报时间 = 周三 |
| **情绪状态** | "这周太累了" | 情绪 = 疲惫 |
| **待办事项** | "下周跟进美团项目" | 待办 = 美团项目 |

### 2.2 智能记忆检索

当用户提问时，Dazee 会**自动关联相关记忆**：

```
用户: "永辉的合同签了吗？金额多少？"

Dazee 内部流程:
1. 检索关键词: "永辉"、"合同"、"金额"
2. 匹配记忆: 
   - "已签署永辉超市合同，金额150万"
   - "周四成功签约"
   - "老张是采购部关键人物"
3. 组合回答: "签了！150万。老张那边也搞定了。"
```

### 2.3 个性化回答

基于用户画像，Dazee 的回答会有针对性：

| 场景 | 普通 AI 回答 | Dazee 个性化回答 |
|------|-------------|-----------------|
| 询问周报时间 | "请问您通常什么时候写周报？" | "你一般**周三**写周报，这周有好消息可以写了！" |
| 工作回顾 | "请提供更多上下文" | "这周重点：永辉合同签署成功(150万)，下周跟进美团项目" |
| 状态询问 | "我不了解您的状态" | "这周虽然累，但成果不错——签下了永辉，老板很满意！" |

### 2.4 智能提醒（规划中）

Dazee 可以基于记忆主动提醒用户：

- ⏰ **截止日期提醒**: "老板要求的永辉合同本周必须签"
- 📅 **习惯性任务**: "今天周三，该写周报了"
- 🎯 **目标追踪**: "美团项目还没跟进，需要安排一下吗？"

## 三、数据输入与输出

### 3.1 输入：用户对话

Mem0 的输入来源是**用户与 Dazee 的自然对话**，无需特殊格式：

```
输入示例（用户消息）:
┌──────────────────────────────────────────────────────────┐
│ "老板刚才在晨会上说，这周必须把永辉超市的合同签下来，   │
│  不然季度KPI完不成"                                      │
└──────────────────────────────────────────────────────────┘
```

### 3.2 输出：结构化记忆

Mem0 自动提取并存储以下结构化数据：

```json
{
  "提取记忆": [
    "本周必须签署永辉超市合同，否则影响季度KPI",
    "老板在晨会上布置任务"
  ],
  "任务线索": {
    "内容": "签署永辉超市的商业合同",
    "截止": "本周",
    "优先级": "高"
  },
  "情绪线索": {
    "信号": "stressed",
    "压力来源": "KPI 压力"
  },
  "关系线索": {
    "提到人物": ["老板"],
    "关系类型": "supervisor"
  }
}
```

### 3.3 数据存储

| 存储层 | 说明 | 持久化 |
|-------|------|--------|
| 向量数据库（腾讯云） | 存储语义化的记忆片段 | ✅ 永久 |
| 用户画像 | 聚合后的用户特征 | ✅ 永久 |
| 会话缓存 | 当前对话的临时信息 | 会话结束后清理 |

## 四、典型使用场景


## 五、与竞品对比

### Dazee Mem0 vs 传统 AI 记忆

| 维度 | 传统方案 | Dazee Mem0 |
|------|---------|-----------|
| **记忆方式** | 关键词匹配 | LLM 语义理解 + 向量检索 |
| **提取粒度** | 需用户明确告知 | 从碎片对话自动推断 |
| **存储** | 会话级（重启丢失） | 用户级（永久保存） |
| **个性化** | 需手动配置偏好 | 自动学习用户习惯 |
| **主动性** | 被动等待用户提问 | 可主动提醒、建议 |

### Dazee Mem0 vs Tolan（行业参考）

> 注：Tolan 是行业内类似的 AI 记忆解决方案

| 维度 | Tolan | Dazee Mem0 |
|------|-------|-----------|
| **目标用户** | 通用消费者 | 企业级专业用户 |
| **记忆重点** | 个人偏好、兴趣 | 工作任务、人物关系、项目进展 |
| **分析深度** | 基础标签提取 | 5W1H 行为模式 + PDCA 计划管理 |
| **情绪关怀** | ❌ 无 | ✅ 情绪追踪 + 时间窗口分析 |
| **汇报生成** | ❌ 无 | ✅ 自动日报/周报生成 |
| **向量存储** | 第三方云服务 | 腾讯云向量数据库（国内合规） |

## 八、常见问题

### Q1: 用户隐私如何保障？

- 数据存储在**腾讯云向量数据库**，符合国内数据合规要求
- 每个用户的记忆**隔离存储**，按 `user_id` 严格区分
- 支持用户**主动删除**个人记忆

### Q2: 记忆会无限增长吗？

- 系统会自动进行**记忆去重和合并**
- 旧的、不再相关的记忆会被**逐步淡化**
- 用户可主动清理特定时间段的记忆

### Q3: 响应速度会变慢吗？

- 记忆检索平均耗时 < 300ms
- 未来将支持**异步存储**，进一步降低用户感知延迟

### Q4: 支持哪些部署方式？

- 当前：腾讯云向量数据库 + 私有部署
- 规划：支持 Qdrant、Chroma 等开源方案
