【跨会话记忆验证 — 茶文化写作提示】

这是一个简单的写作任务提示，用于验证小搭子是否在新会话中自动应用了上一次学到的写作风格。

---

用户输入（会话2 轮次1）：

"写篇关于茶文化的文章，大概1500字"

---

【预期结果对比】

如果记忆生效（小搭子预期）：
- 自动应用"毒舌+干货"风格
- 开头用反讽或打破常规认知的方式切入
- 文章中有数据支撑
- 末尾有"干货时间"板块
- 口语化表达，多短句和反问
- 用户无需再次说明风格偏好

如果记忆未生效（clawdbot 可能情况）：
- 输出四平八稳的通用文章
- 正式语气，无个人风格
- 需要用户再次说明"我要毒舌+干货风格"

---

【Token 消耗对比要点】

小搭子：
- recall() 从 FTS5 + Mem0 检索风格记忆 → ~200ms，0 额外 LLM 调用
- 记忆内容注入到 context injector → 作为 messages 层注入，不影响 system prompt 缓存
- 预估额外 Token：~200-300 tokens（记忆内容）

clawdbot：
- memory_search 语义搜索 → 召回记忆
- 记忆内容注入到 context → 可能影响缓存
- 如果召回失败 → 需要额外 1-2 轮对话确认风格 → 额外 ~3000-5000 tokens
