## 二、延迟加载机制

### 2.1 设计原理

**问题：** 全量加载 52 个 Skills 到系统 Prompt 导致：
- 系统 Prompt 过长（~147K 字符）
- 每次对话重复传输所有 Skills
- Token 成本高昂

**解决方案：** 借鉴 clawdbot 的延迟加载策略
- 只在系统 Prompt 中注入 **Skills 列表**（name + description + location）
- Agent 根据任务选择相关 Skill 后，通过 **Read 工具**读取完整 SKILL.md
- 按需加载，避免 Token 浪费

### 2.2 实现

#### 核心模块：`core/prompt/skill_prompt_builder.py`

```python
class SkillPromptBuilder:
    """Skill Prompt 构建器"""
    
    @staticmethod
    def build_lazy_prompt(
        skills: List[SkillSummary],
        language: str = "zh",
    ) -> str:
        """构建延迟加载 Prompt（仅注入 Skills 列表）"""
        lines = ["<available_skills>"]
        
        for skill in skills:
            lines.append(f'  <skill name="{skill.name}" location="{skill.location}">')
            lines.append(f'    <description>{skill.description}</description>')
            lines.append('  </skill>')
        
        lines.append("</available_skills>")
        return "\n".join(lines)
```

#### 注入格式示例

```xml
<available_skills>
  <skill name="github" location="/path/to/skills/github/SKILL.md">
    <description>Interact with GitHub using the `gh` CLI...</description>
  </skill>
  <skill name="gemini" location="/path/to/skills/gemini/SKILL.md">
    <description>Gemini CLI for one-shot Q&A...</description>
  </skill>
</available_skills>
```

#### 系统指令

```markdown
## Skills (mandatory)

Scan `<available_skills>` `<description>` entries.
- If exactly one skill clearly applies: read its SKILL.md with Read, then follow it.
- If multiple could apply: choose the most specific one, then read/follow it.
- If none clearly apply: do not read any SKILL.md.

**Constraints:** 
- Only read SKILL.md when clearly needed
- Never read more than one skill up front
- Use Read tool with the `location` path
```

### 2.3 效果验证

**测试结果（真实数据）：**

|------|-------------|-----------|----------|
| 全量加载 | 146,891 字符 | ~36,723 tokens | - |
| 延迟加载 | 15,372 字符 | ~3,843 tokens | **89.5%** |

**成本估算：**

- **每次对话节省**：$0.11 USD
- **年度节省**（1000 次/天）：**$40,332.50 USD**

## 三、依赖检查集成

### 3.1 设计原理

**问题：** Skills 依赖检查与实例加载分离，用户体验差：
- 用户需手动运行 `check_instance_dependencies.py`
- 启动时不知道哪些 Skills 缺少依赖
- 部署态缺少引导

**解决方案：** 将依赖检查集成到 `instance_loader`：
- 启动时自动检查依赖
- 支持多种模式：提示（prompt）、自动安装（auto）、交互式（interactive）
- 配置化控制（config.yaml）

### 3.2 实现

#### 集成到 `instance_loader.py`

```python
async def create_agent_from_instance(
    instance_name: str,
    # ... 其他参数
    check_dependencies: bool = True,
    dependency_mode: str = "prompt",  # prompt / auto / interactive / skip
):
    # 1. 加载环境变量
    load_instance_env(instance_name)
    
    # 🆕 V6.2: Skill 依赖检查
    if check_dependencies and dependency_mode != "skip":
        await _check_skill_dependencies(instance_name, dependency_mode)
    
    # 2. 继续加载实例...
```

#### 依赖检查实现

```python
async def _check_skill_dependencies(instance_name: str, mode: str = "prompt"):
    """检查 Skill 依赖"""
    from core.skill.dynamic_loader import DependencyChecker
    
    checker = DependencyChecker(instance_name)
    results = checker.check_all_skills()
    
    if results['missing']:
        if mode == "prompt":
            # 仅提示
            logger.warning(f"⚠️ {len(results['missing'])} 个 Skills 缺少依赖")
            logger.info("运行以下命令查看详情：")
            logger.info(f"  python scripts/check_instance_dependencies.py {instance_name}")
        
        elif mode == "auto":
            # 自动安装
            logger.info(f"📦 自动安装 {len(results['missing'])} 个 Skills 依赖...")
            script = checker.generate_install_script(results)
            await run_install_script_async(script)
        
        elif mode == "interactive":
            # 交互式引导
            checker.run_interactive(results)
```

### 3.3 使用方式

#### 启动模式

```bash
# 模式 1: 仅提示（默认）
python scripts/run_instance.py client_agent
# 输出: ⚠️ 30 个 Skills 缺少依赖，运行 check_instance_dependencies.py 查看详情

# 模式 2: 交互式引导
python scripts/run_instance.py client_agent --dependency-mode interactive
# 输出: 
#   📦 peekaboo 需要安装: brew install steipete/tap/peekaboo
#   是否安装? [Y/n]

# 模式 3: 自动安装（未来支持）
python scripts/run_instance.py client_agent --dependency-mode auto
# 输出: 
#   📦 正在安装 peekaboo...
#   ✅ 安装成功
```

#### 配置控制（config.yaml）

```yaml
# ==================== Skills 依赖管理配置 ====================
skill_dependency_check:
  enabled: true                # 是否启用依赖检查
  mode: "prompt"               # 检查模式：prompt / auto / interactive / skip
  fail_on_missing: false       # 缺少依赖时是否阻止启动
  install_preferences:
    prefer_brew: true          # 优先使用 Homebrew
    node_manager: "npm"        # npm/pnpm/yarn/bun
```
