# V9.0 意图识别优化总结

## 实施日期
2026-01-24

## 核心问题

### 问题 1：上下文污染
**现象**：意图识别时，完整对话历史（含大量工具调用结果、代码块、长文本回复）会污染上下文，影响意图准确率。

**根因**：意图识别的本质是理解用户 query，与智能体的执行细节关联性不高。

### 问题 2：意图状态管理缺失
**现象**：用户追问时，系统重新分析意图但未复用上轮的 plan 和 execution_context。

**根因**：缺少意图状态转换机制（新意图 vs 追问）。

---

## 解决方案

### 1. 用户问题聚焦型过滤（Intent-Focused Context Filter）

**实现位置**：
- `core/routing/intent_analyzer.py`（主要实现）
- `core/agent/intent_analyzer.py`（已删除，冗余副本）

**核心方法**：
```python
def _filter_for_intent(self, messages: List[Dict]) -> List[Dict]:
    """
    O(n) 纯 CPU 过滤，< 0.1ms
    
    规则：
    1. 只保留最近 5 条 role=user 消息
    2. 最后 1 条 role=assistant 截断为 100 字符
    3. 直接丢弃 tool_use/tool_result
    """
```

**时延保证**：
- 过滤逻辑：< 0.1ms（纯 CPU）
- 总时延：< 200ms（含 LLM 调用）

**收益**：
- Token 消耗：~5-10K → ~0.5-1K
- 意图准确率：减少噪音，更精准

### 2. 意图状态转换（Intent State Transition）

**实现位置**：
- `core/agent/simple/simple_agent.py::_handle_intent_transition()`

**核心逻辑**：
```python
def _handle_intent_transition(self, new_intent: IntentResult):
    if new_intent.is_follow_up and previous_intent:
        # 追问：复用 plan_cache，继承 task_type
        logger.info("🔄 追问模式: 复用 plan_cache")
        new_intent.task_type = previous_intent.task_type
        # plan_cache 保持不变
    else:
        # 新意图：重置 plan_cache
        logger.info("🆕 新意图: 重置 plan_cache")
        self._plan_cache = {"plan": None, "todo": None, "tool_calls": []}
```

**状态转换表**：

| 转换类型 | 触发条件 | 状态处理 |
|----------|----------|----------|
| 追问 | `is_follow_up=true` | 保留 plan_cache，继承 task_type |
| 新意图 | `is_follow_up=false` | 重置 plan_cache，清空 tool_calls |
