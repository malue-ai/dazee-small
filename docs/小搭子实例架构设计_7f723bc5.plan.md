# 小搭子专用实例架构设计

## 一、架构定位

基于 ZenFlux Agent V9.0，为"养在电脑里的小搭子"设计专用桌面端实例架构。

### 核心差异化


| 维度    | ZenFlux（云端）       | 小搭子（桌面端）                      |
| ----- | ----------------- | ----------------------------- |
| 工具机制  | Tools + Skills 混合 | **Skills-First**（对齐 OpenClaw） |
| 终止策略  | 固定轮数限制            | **自适应终止**（Agent 自主决定）         |
| 运行环境  | 服务器               | **桌面端 Tauri**（跨 OS）           |
| UI 交互 | SSE 文本            | **MCP Apps 交互式 UI**           |

## 三、核心机制设计

### 3.1 零配置/低配置设计（核心用户体验）

**核心问题**：普通用户不懂技术配置，如何让小搭子"开箱即用"？

**设计原则**：

1. **零配置可用**：首次启动即可使用基础功能
2. **渐进式配置**：先用起来，再逐步解锁高级功能
3. **可视化引导**：清晰的 UI 界面指导配置过程
4. **智能诊断**：自动检测问题并提供解决方案

#### 3.1.1 Skills 二维分类体系（OS × 依赖复杂度）

**设计原则**：每个 Skill 同时具备两个属性维度，统一管理

**依赖复杂度定义**：


| 等级            | 名称  | 说明            | 用户操作      |
| ------------- | --- | ------------- | --------- |
| `builtin`     | 内置  | 小搭子安装包已包含     | 无需操作      |
| `lightweight` | 轻量  | Python 包或系统授权 | 自动处理或一键授权 |
| `external`    | 外部  | 需要安装外部应用/工具   | 提示安装指南    |
| `cloud_api`   | 云服务 | 需要 API Key    | 配置向导      |


**二维矩阵概览**：

```
                    依赖复杂度 →
           ┌──────────┬─────────────┬──────────┬───────────┐
           │ builtin  │ lightweight │ external │ cloud_api │
    OS ↓   │ (内置)    │ (轻量)      │ (外部)    │ (云服务)   │
┌──────────┼──────────┼─────────────┼──────────┼───────────┤
│ common   │summarize │excel-       │obsidian  │notion     │
│ (跨平台)  │canvas    │analyzer     │          │trello     │
│          │translator│nano-pdf     │          │gemini-img │
├──────────┼──────────┼─────────────┼──────────┼───────────┤
│ darwin   │screenshot│apple-notes  │peekaboo  │           │
│ (macOS)  │clipboard │apple-remind │bear-notes│           │
├──────────┼──────────┼─────────────┼──────────┼───────────┤
│ win32    │screenshot│outlook-cli  │powershell│           │
│ (Windows)│clipboard │onenote      │(admin)   │           │
├──────────┼──────────┼─────────────┼──────────┼───────────┤
│ linux    │screenshot│notify-send  │xdotool   │           │
```

**统一配置（合并 OS 分层和依赖分层）**：

```yaml
# instances/xiaodazi/config.yaml

skills:
  loading_mode: "lazy"
  os_aware: true
  
  # ========== 跨平台 Skills ==========
  common:
    builtin:
      - summarize
      - canvas
      - translator
      - writing-assistant
      - file-manager
    lightweight:
      excel-analyzer:
        python_packages: ["pandas", "openpyxl"]
        auto_install: true
      nano-pdf:
        python_packages: ["pypdf"]
        auto_install: true
    external:
      obsidian:
        requires_app: "Obsidian"
        detect_path:
          darwin: "~/Library/Application Support/obsidian"
          win32: "%APPDATA%/obsidian"
          linux: "~/.config/obsidian"
    cloud_api:
      notion:
        api_key_field: "notion_api_key"
        config_wizard: "notion_setup"
      gemini-imagen:
        api_key_field: "gemini_api_key"  # 复用用户的 Gemini Key
        
  # ========== macOS 专属 ==========
  darwin:
    builtin:
      - macos-screenshot    # screencapture
      - macos-clipboard     # pbcopy/pbpaste
      - macos-notification  # osascript
      - applescript
    lightweight:
      apple-notes:
        system_auth: "accessibility"
        auth_reason: "需要控制 Notes 应用"
        auth_guide: "/guides/macos-accessibility"
      apple-reminders:
        system_auth: "reminders"
      apple-calendar:
        system_auth: "calendar"
    external:
      peekaboo:
        bins: ["peekaboo"]
        install:
          brew: "brew install peekaboo"
      bear-notes:
        requires_app: "Bear"
        
  # ========== Windows 专属 ==========
  win32:
    builtin:
      - windows-screenshot
      - windows-clipboard
      - windows-notification
      - powershell-basic
    lightweight:
      outlook-cli:
        requires_app: "Microsoft Outlook"
      onenote:
        requires_app: "OneNote"
    external:
      powershell-admin:
        requires: "administrator"
        
  # ========== Linux 专属 ==========
  linux:
    builtin:
      - linux-screenshot
      - linux-clipboard
      - linux-notification
    lightweight:
      notify-send:
        bins: ["notify-send"]
        install:
          apt: "apt install libnotify-bin"
          yum: "yum install libnotify"
    external:
      xdotool:
        bins: ["xdotool"]
        install:
          apt: "apt install xdotool"
```

```
│    └──────────────────────────────────────────────────────┘ │
└──────────────────────────────────────────────────────────────┘
    │
    ▼

│    {                                                         │
│      "summarize":    { status: "ready" },                   │
│      "apple-notes":  { status: "need_auth", auth: "..." },  │

│      "peekaboo":     { status: "unavailable", hint: "..." },│

└──────────────────────────────────────────────────────────────┘
    │
    ▼
用户使用时
    │
    ├─ ready       → 直接执行
    ├─ need_auth   → 弹出授权向导 → 授权后执行
    ├─ need_setup  → 弹出 API Key 配置向导 → 配置后执行
    └─ unavailable → 提示原因 + 安装指南 / 替代方案
```

### 3.3 状态一致性管理（核心设计）

**问题**：桌面端任务涉及文件操作、应用状态修改，中途中断可能导致：

- 文件被部分修改（半成品状态）
- 应用状态不一致
- 用户数据丢失或损坏

**设计原则**：任务要么完整完成，要么能够安全回滚到初始状态

#### 状态一致性生命周期

```
任务开始
    │
    ▼
┌─────────────────────────────────────────────────────────────┐
│ 1. 环境快照（Snapshot）                                       │
│    • 记录当前工作目录                                          │
│    • 备份将要修改的文件（增量备份）                              │
│    • 记录剪贴板内容                                            │
│    • 记录相关应用状态                                          │
└─────────────────────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────────────────────┐
│ 2. 操作日志（Operation Log）                                   │
│    每个操作记录：                                               │
│    • operation_id: 唯一标识                                    │
│    • action: 具体操作（write_file, delete, rename...）         │
│    • target: 操作目标（文件路径、应用名）                        │
│    • before_state: 操作前状态                                  │
│    • after_state: 操作后状态                                   │
│    • rollback_action: 逆操作（用于回滚）                        │
│    • timestamp: 时间戳                                         │
└─────────────────────────────────────────────────────────────┘
    │
    ├─── 正常完成 ───────────────────────────────────────────────┐
    │                                                            │
    │    ┌────────────────────────────────────────────────┐      │
```

#### 状态管理器核心实现

```python
# core/state/consistency_manager.py

from dataclasses import dataclass
from typing import List, Optional
from pathlib import Path
import shutil
import json

@dataclass
class OperationRecord:
    """单个操作记录"""
    operation_id: str
    action: str                    # write_file, delete, rename, ...
    target: str                    # 文件路径或资源标识
    before_state: Optional[dict]   # 操作前状态
    after_state: Optional[dict]    # 操作后状态
    rollback_action: dict          # 逆操作定义
    timestamp: float
    
class StateConsistencyManager:
    """
    状态一致性管理器
    
    职责：
    1. 任务开始前创建快照
    2. 记录每个操作的日志和逆操作
    3. 异常时提供回滚能力
    4. 任务完成后清理快照
    """
    
    def __init__(self, config: dict):
        self.config = config
        self.snapshot_path = Path(config.get("snapshot_path", "~/.xiaodazi/snapshots")).expanduser()
        self.current_snapshot_id: Optional[str] = None
        self.operation_log: List[OperationRecord] = []
        
    async def create_snapshot(self, task_id: str, affected_files: List[Path]) -> str:
        """
        创建任务快照
        
        Args:
            task_id: 任务 ID
            affected_files: 将要修改的文件列表
            
        Returns:
            snapshot_id: 快照 ID
        """
        snapshot_id = f"snap_{task_id}_{int(time.time())}"
        snapshot_dir = self.snapshot_path / snapshot_id
        snapshot_dir.mkdir(parents=True, exist_ok=True)
        
        # 备份将要修改的文件
        for file_path in affected_files:
            if file_path.exists():
                backup_path = snapshot_dir / file_path.name
                shutil.copy2(file_path, backup_path)
                
        # 记录环境状态
        env_state = {
            "cwd": str(Path.cwd()),
            "clipboard": await self._capture_clipboard(),
            "timestamp": time.time(),
            "files": [str(f) for f in affected_files]
        }
        
        with open(snapshot_dir / "env_state.json", "w") as f:
            json.dump(env_state, f, ensure_ascii=False, indent=2)
            
        self.current_snapshot_id = snapshot_id
        logger.info(f"快照创建完成: {snapshot_id}, 备份文件数: {len(affected_files)}")
        
        return snapshot_id
        
    def record_operation(self, record: OperationRecord):
        """记录操作（用于回滚）"""
        self.operation_log.append(record)
        logger.debug(f"操作记录: {record.action} -> {record.target}")
        
    async def rollback(self, to_snapshot_id: Optional[str] = None) -> bool:
        """
        回滚到指定快照
        
        Args:
            to_snapshot_id: 目标快照 ID，None 则回滚到当前任务开始前
            
        Returns:
            是否回滚成功
        """
        snapshot_id = to_snapshot_id or self.current_snapshot_id
        if not snapshot_id:
            logger.warning("没有可用的快照")
            return False
            
        logger.info(f"开始回滚到快照: {snapshot_id}")
        
        # 1. 逆序执行回滚操作
        for record in reversed(self.operation_log):
            try:
                await self._execute_rollback_action(record.rollback_action)
                logger.debug(f"回滚操作成功: {record.action} -> {record.target}")
            except Exception as e:
                logger.error(f"回滚操作失败: {record.action}, 错误: {e}")
                
        # 2. 恢复快照中的文件
        snapshot_dir = self.snapshot_path / snapshot_id
        if snapshot_dir.exists():
            env_state_file = snapshot_dir / "env_state.json"
            if env_state_file.exists():
                with open(env_state_file) as f:
                    env_state = json.load(f)
                    
                for file_name in snapshot_dir.iterdir():
                    if file_name.name != "env_state.json":
                        # 恢复原始文件
                        original_path = self._find_original_path(file_name.name, env_state)
                        if original_path:
                            shutil.copy2(file_name, original_path)
                            logger.info(f"文件已恢复: {original_path}")
                            
        # 3. 清理操作日志
        self.operation_log.clear()
        
        logger.info(f"回滚完成: {snapshot_id}")
        return True
        
    async def commit(self):
        """
        提交任务（清理快照）
        
        任务成功完成后调用
        """
        if self.current_snapshot_id:
            snapshot_dir = self.snapshot_path / self.current_snapshot_id
            if snapshot_dir.exists():
                shutil.rmtree(snapshot_dir)
                logger.info(f"快照已清理: {self.current_snapshot_id}")
                
        self.operation_log.clear()
        self.current_snapshot_id = None
        
    async def get_rollback_options(self) -> dict:
        """
        获取回滚选项（用于 HITL UI）
        
        Returns:
            回滚选项信息
        """
        return {
            "has_snapshot": self.current_snapshot_id is not None,
            "operations_count": len(self.operation_log),
            "modified_files": [r.target for r in self.operation_log if r.action in ["write_file", "delete"]],
            "options": [
                {"id": "rollback", "label": "回滚到任务开始前的状态", "description": "撤销所有修改"},
                {"id": "keep", "label": "保持当前状态", "description": "可能存在不完整的修改"},
                {"id": "retry", "label": "尝试继续完成", "description": "从中断点继续"}
            ]
        }
```

### 3.4 多维度终止策略

**问题**：固定 `max_turns` 限制不适合桌面端场景

- 简单任务可能 2 轮就完成
- 复杂任务可能需要 50+ 轮
- 需要支持用户随时干预

**解决方案**：多维度终止策略（LLM 自主 + HITL + 用户停止 + 安全兜底）

#### 配置项（续）

```yaml
  # ==================== 多维度终止策略 ====================
  termination:
    mode: "adaptive"              # adaptive / fixed / hybrid
    
    # ==================== 1. LLM 自主终止 ====================
    llm_self_termination:
      enabled: true
      # LLM 在每轮结束时自评是否完成
      self_assessment: true
      
      # 任务完成信号词
      completion_signals:
        - "任务已完成"
        - "已为您"
        - "完成了"
        - "请查看结果"
      
      # 用户满意度检测
      satisfaction_check: true
      
    # ==================== 2. HITL 人工干预（集成状态一致性）====================
    hitl:
      enabled: true
      
      # 危险操作必须确认
      require_confirmation:
        - "delete"                # 删除操作
        - "overwrite"             # 覆盖操作
        - "send_email"            # 发送邮件
        - "publish"               # 发布内容
        - "payment"               # 支付相关
        
      # HITL 触发后暂停等待
      wait_timeout_seconds: 300   # 等待用户响应 5 分钟
      
      # 用户拒绝后的处理（集成回滚）
      on_rejection: "ask_rollback" # stop / rollback / ask_rollback / ask_alternative
      
      # 异常/中断时的处理
      on_error:
        show_rollback_options: true   # 显示回滚选项
        auto_rollback_critical: true  # 严重错误自动回滚
        
    # ==================== 3. 用户主动停止 ====================
    user_stop:
      enabled: true
      
      # 停止信号识别
      stop_keywords:
        - "停止"
        - "算了"
        - "取消"
        - "不用了"
        - "stop"
        - "cancel"
        
      # 停止后动作（集成回滚选项）

        summarize: true           # 总结当前进度
        save_state: true          # 保存状态，可恢复
        offer_rollback: true      # 询问是否需要回滚

    # ==================== 4. 安全兜底 ====================
    safety_limits:
      max_turns: 100              # 绝对上限
      max_duration_minutes: 30    # 时间上限
      idle_timeout_seconds: 60    # 无活动超时
      max_consecutive_failures: 5 # 连续失败上限
      
    # ==================== 5. 长任务用户确认 ====================

      enabled: true
      confirm_threshold: 20       # 超过 N 轮时询问
      confirm_message: "任务已执行 {turns} 轮，是否继续？"
      options:
        - "继续执行"
        - "暂停并总结"
        - "停止任务"
```

```python
class OSCompatibilityChecker:
    """检查 Skill 是否兼容当前 OS"""
    
    def check_skill(self, skill_metadata: dict) -> CompatibilityResult:
        """
        检查 Skill 兼容性
        
        Returns:
            CompatibilityResult:
                - compatible: bool
                - missing_deps: List[str]
                - alternatives: List[str]  # 替代 Skill
        """
        current_os = platform.system().lower()
        os_map = {"darwin": "darwin", "windows": "win32", "linux": "linux"}
        
        skill_os = skill_metadata.get("openclaw", {}).get("os", [])
        
        if os_map.get(current_os) not in skill_os:
            return CompatibilityResult(
                compatible=False,
                reason=f"Skill 不支持 {current_os}",
                alternatives=self._find_alternatives(skill_metadata)
            )
        
        # 检查依赖
        required_bins = skill_metadata.get("requires", {}).get("bins", [])
        missing = [b for b in required_bins if not shutil.which(b)]
        
        return CompatibilityResult(
            compatible=len(missing) == 0,
            missing_deps=missing
        )
```

#### 3.5.4 进度转换器（核心实现）

```python
# core/planning/progress_transformer.py

class ProgressTransformer:
    """
    将技术性的 plan_todo 步骤转换为用户友好的进度消息
    
    设计原则：
    - 内部保留完整的规划和回溯能力
    - 对用户只暴露简化的进度反馈
    """
    
    def __init__(self, config: dict):
        self.ui_mode = config.get("ui_mode", "progress_bar")
        self.progress_messages = config.get("progress_messages", {})
    
    def transform(self, plan_step: dict) -> ProgressUpdate:
        """
        转换计划步骤为用户友好的进度消息
        
        Args:
            plan_step: plan_todo 的原始步骤
            
        Returns:
            ProgressUpdate: 用户友好的进度信息
        """
        action = plan_step.get("action", "")
        
        # 动作到友好消息的映射
        message_map = {
            "read_file": self.progress_messages.get("analyzing", "正在分析..."),
            "fix_format": self.progress_messages.get("fixing", "自动修复中..."),
            "process_data": self.progress_messages.get("processing", "处理中..."),
            "generate_output": self.progress_messages.get("almost_done", "快好了..."),
        }
        
        friendly_message = message_map.get(action, "正在处理...")
        
        return ProgressUpdate(
            message=friendly_message,
            percentage=plan_step.get("progress", 0),
            show_details=False  # 默认不显示技术细节
        )
```

#### 技术实现

```python
# core/knowledge/local_search.py

class LocalKnowledgeManager:
    """本地知识管理器（以文件为中心）"""
    
    def __init__(self, config: KnowledgeConfig):
        self.config = config
        self.fulltext_engine = SQLiteFTS5Engine()  # 内置，零依赖
        self.semantic_engine = None  # 延迟加载
        
    async def search(self, query: str) -> List[SearchResult]:
        """智能搜索：优先语义，降级全文"""
        if self.config.semantic.enabled and self._has_embedding_api():
            return await self._semantic_search(query)
        return await self._fulltext_search(query)
        
    async def _fulltext_search(self, query: str) -> List[SearchResult]:
        """全文搜索（零配置）"""
        # SQLite FTS5，无需任何外部依赖
        return await self.fulltext_engine.search(query)
        
    async def _semantic_search(self, query: str) -> List[SearchResult]:
        """语义搜索（复用用户 LLM API）"""
        if not self.semantic_engine:
            self.semantic_engine = await self._init_semantic_engine()
        return await self.semantic_engine.search(query)
        
    def _has_embedding_api(self) -> bool:
        """检查用户是否已配置可用的 embedding API"""
        # 复用用户配置的 Gemini/OpenAI，不额外引入配置
        return bool(self.config.llm.gemini_api_key or 
                    self.config.llm.openai_api_key)
```

```python
# core/discovery/app_scanner.py

class AppScanner:
    """发现用户已安装的应用，用于精准推荐"""
    
    async def scan(self) -> List[InstalledApp]:
        """扫描已安装应用"""
        os_type = platform.system().lower()
        
        if os_type == "darwin":
            return await self._scan_darwin()
        elif os_type == "windows":
            return await self._scan_win32()
        else:
            return await self._scan_linux()
            
    async def _scan_darwin(self) -> List[InstalledApp]:
        """macOS 应用扫描"""
        apps = []
        
        # 1. 扫描 /Applications
        for app_path in Path("/Applications").glob("*.app"):
            apps.append(self._parse_macos_app(app_path))
            
        # 2. brew cask 列表
        brew_apps = await self._run_cmd("brew list --cask")
        apps.extend(self._parse_brew_list(brew_apps))
        
        return apps
        
    def get_capabilities(self, app_name: str) -> List[str]:
        """获取应用的能力标签"""
        return self.capability_mapping.get(app_name, [])
        
    def find_app_for_task(self, task: str) -> Optional[InstalledApp]:
        """为任务找到合适的已安装应用"""
        # 例：task = "打开流程图" → 找到 Visio 或 draw.io
        required_capability = self._task_to_capability(task)
        
        for app in self.installed_apps:
            if required_capability in app.capabilities:
                return app
        return None
```

#### 核心组件实现

```python
# core/memory/xiaodazi_memory.py

class XiaodaziMemoryManager:
    """小搭子记忆管理器（三层架构）"""
    
    def __init__(self, config: MemoryConfig):
        self.config = config
        self.file_layer = MarkdownMemoryLayer(config.files)
        self.index_layer = SQLiteIndexLayer(config.index)
        self.smart_layer = MemoryExtractor(config.extraction)
        
    # ========== 读取记忆 ==========
    
    async def recall(self, query: str, project_id: str = None) -> List[Memory]:
        """回忆相关记忆"""
        # 1. 混合搜索
        if self.config.index.hybrid.enabled:
            results = await self._hybrid_search(query, project_id)
        else:
            results = await self._fulltext_search(query, project_id)
            
        # 2. 重排序
        results = self._rerank(results, query)
        
        return results
        
    async def _hybrid_search(self, query: str, project_id: str) -> List[Memory]:
        """混合搜索：BM25 + 向量"""
        # BM25 全文搜索
        bm25_results = await self.index_layer.fts_search(query)
        
        # 向量搜索（如果启用）
        if self.config.index.vector.enabled:
            vec_results = await self.index_layer.vector_search(query)
            return self._merge_results(bm25_results, vec_results)
        
        return bm25_results
        
    # ========== 写入记忆 ==========
    
    async def remember(self, content: str, category: str, project_id: str = None):
        """记住新信息"""
        # 1. 冲突检测
        if self.config.extraction.quality.conflict_detection:
            conflicts = await self._detect_conflicts(content)
            if conflicts:
                await self._resolve_conflicts(content, conflicts)
                
        # 2. 写入 Markdown
        await self.file_layer.append_memory(content, category, project_id)
        
        # 3. 更新索引
        await self.index_layer.index_memory(content, category, project_id)
        
    # ========== 自动提取 ==========
    
    async def extract_from_conversation(self, messages: List[Message]):
        """从对话中自动提取记忆"""
        # 使用 LLM 提取（借鉴 mem0）
        extracted = await self.smart_layer.extract(messages)
        
        for memory in extracted:
            if memory.confidence >= self.config.extraction.quality.confidence_threshold:
                await self.remember(
                    content=memory.content,
                    category=memory.category,
                    project_id=memory.project_id
                )
                
    # ========== 会话结束刷新 ==========
    
    async def flush(self, session_id: str, messages: List[Message]):
        """会话结束时刷新记忆"""
        # 1. 提取新记忆
        await self.extract_from_conversation(messages)
        
        # 2. 写入每日日志
        today = datetime.now().strftime("%Y-%m-%d")
        log_path = f"memory/{today}.md"
        
        summary = await self._summarize_session(messages)
        await self.file_layer.append_log(log_path, summary)
```

### 6.2 OS 分层 Skills 合并器核心代码

```python
# core/skill/os_skill_merger.py

import platform
from typing import Dict, Set

class OSSkillMerger:
    """
    OS 分层 Skills 合并器
    
    职责：根据当前 OS 合并 common + OS 专属 Skills
    """
    
    OS_MAP = {
        "Darwin": "darwin",
        "Windows": "win32",
        "Linux": "linux"
    }
    
    def __init__(self, config: dict):
        self.config = config
        self.current_os = self.OS_MAP.get(platform.system(), "linux")
    
    def get_enabled_skills(self) -> Dict[str, int]:
        """
        获取当前 OS 的启用 Skills
        
        Returns:
            合并后的 Skills 字典 {skill_name: enabled}
        """
        # 1. 获取跨平台 Skills
        common_skills = self.config.get("skills_common", {})
        
        # 2. 获取 OS 专属 Skills
        os_key = f"skills_{self.current_os}"
        os_skills = self.config.get(os_key, {})
        
        # 3. 合并（OS 专属优先）
        merged = {**common_skills, **os_skills}
        
        # 4. 过滤启用的
        enabled = {k: v for k, v in merged.items() if v == 1}
        
        logger.info(
            f"Skills 合并完成: OS={self.current_os}, "
            f"common={len(common_skills)}, "
            f"os_specific={len(os_skills)}, "
            f"total_enabled={len(enabled)}"
        )
        
        return enabled
    
    def get_unavailable_skills(self) -> Set[str]:
        """
        获取当前 OS 不可用的 Skills（其他 OS 专属）
        
        用于提示用户某些功能在当前平台不可用
        """
        all_os_keys = ["skills_darwin", "skills_win32", "skills_linux"]
        current_key = f"skills_{self.current_os}"
        
        unavailable = set()
        for key in all_os_keys:
            if key != current_key:
                unavailable.update(self.config.get(key, {}).keys())
        
        return unavailable
```
