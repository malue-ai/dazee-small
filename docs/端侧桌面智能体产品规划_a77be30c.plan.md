# 养在电脑里的小搭子 - 产品规划

## 一、产品定位

### 口号

**"养在电脑里的小搭子"**

副标语：**用得越久，越懂你 | 说一句，它就干**

### 核心理念：不只是聊天，是真能干活的搭子

```
传统 AI 助手：               我们的小搭子：
"我来告诉你怎么做..."        "我来帮你干..."
                              
用户还要自己动手              用户只管提需求
```



阿里云



### 三大核心能力


| 能力      | 说明              | 技术支撑               |
| ------- | --------------- | ------------------ |
| **会干活** | 不只是回答问题，真的能执行任务 | 工具调用、沙箱执行、文件操作     |
| **会思考** | 失败了自己想办法，不用你反复教 | RVR-B 智能回溯、错误自愈    |
| **会学习** | 用得越久越懂你的习惯和风格   | 持续学习、项目记忆、Playbook |


### 差异化策略


| 对比维度     | 传统 AI 工具    | 小搭子         |
| -------- | ----------- | ----------- |
| **安装使用** | 配置复杂，技术门槛高  | 双击安装，5 分钟上手 |
| **执行能力** | 只能聊天，不能干活   | 真的能帮你执行任务   |
| **出错处理** | 报错停止，需要人工调试 | 自动修复，智能回溯   |
| **用户记忆** | 每次重新解释      | 越用越懂你       |
| **目标用户** | 开发者或技术用户    | 所有人，尤其非技术用户 |
| **安全性**  | 第三方插件风险     | 官方签名验证      |
| **成本**   | 订阅制或需要服务器   | 本地运行        |


### 目标用户


| 用户群        | 他们要的搭子是...       | 核心场景          |
| ---------- | ---------------- | ------------- |
| **内容创作者**  | 记得我风格、帮我写稿的搭子    | 公众号、视频脚本、文案   |
| **职场人**    | 帮我处理表格、做 PPT 的搭子 | Excel、报告、汇报材料 |
| **研究工作者**  | 帮我整理文献、润色论文的搭子   | 论文、研究、知识管理    |
| **自由职业者**  | 帮我管理多个项目的搭子      | 多客户、多项目切换     |
| **隐私敏感用户** | 数据不出电脑的搭子        | 可选接入本地模型      |

## 二、核心 Use Cases（5 个爆点场景）

### Use Case 1：写稿搭子（内容创作者）

**场景故事**：

```
小雨运营 3 个公众号，每个号风格不同。

之前用 ChatGPT：
"帮我写篇咖啡文化" → 出来一篇四平八稳的文章
"不对，我要毒舌风格" → 改一版
"还是不对，我的风格是毒舌但有干货" → 又改一版
每次都要重新解释...

现在用小搭子：
打开「毒舌咖啡馆」项目
"写篇咖啡文化"
→ 小搭子已学习 127 篇历史文章
→ 直接输出毒舌+干货风格
→ 一稿过！
```

- **核心能力**：持续学习、项目记忆
- **金句**："3 个月后，它比我自己更懂我的写作风格"

### Use Case 2：表格搭子（职场人）

**场景故事**：

```
王姐是财务，经常处理各种格式奇怪的 Excel。

之前用 AI：
"分析这个表格" → "格式错误，无法解析"
"那你转换一下格式" → "转换失败，请检查..."
来回 5 轮，还是自己手动处理了...

现在用小搭子：
拖入 Excel → "帮我分析销售趋势"
→ 小搭子自动检测到日期格式问题，自己修复
→ 检测到合并单元格，自动处理
→ 检测到数据量大，自动采样优化
→ 输出分析报告 + 图表
王姐全程没操心格式问题
```

- **核心能力**：智能回溯、自动纠错
- **金句**："以前我要教 AI 怎么处理错误，现在它自己就搞定了"

### Use Case 3：环境搭子（非技术用户）

**场景故事**：

```
老李是产品经理，不懂技术。

之前用 AI：
"帮我把流程图导出 PNG" → "请用 Figma 打开..."
"我没装 Figma" → "那用 Sketch..."
"我是 Windows" → "那用 draw.io..."
来回好几轮才知道我有 Visio...

现在用小搭子：
首次启动时，小搭子已经知道：
- 老李用 Windows 11
- 装了 Office 全家桶（含 Visio）
- 装了 WPS、腾讯文档

"帮我导出流程图"
→ "检测到您安装了 Visio，我来帮您操作..."
→ 直接给出 Visio 的操作步骤
```

- **核心能力**：环境感知、精准推荐
- **金句**："它住在我电脑里，当然知道我有什么软件"

### Use Case 4：研究搭子（研究工作者）

**场景故事**：

```
小陈是研三学生，正在写毕业论文。

痛点：
- 其他 AI 工具配置太复杂，折腾半天没搞定
- 云端订阅太贵，学生党用不起

现在用小搭子：
下载 → 双击安装 → 打开
选择"使用云端 API"（按量付费，更经济）
→ 5 分钟上手

"帮我润色这段论文" → 学术风格润色
"整理这篇 PDF 的引用" → 自动提取引用格式
```

- **核心能力**：开箱即用、专业场景
- **金句**："双击安装，5 分钟上手，不用折腾"

### Use Case 5：隐私搭子（隐私敏感用户）

**场景故事**：

```
张律师处理很多保密合同，不敢用云端 AI。

小搭子的灵活性：
设置 → 模型配置 → 选择"本地模型（Ollama）"
→ 下载一个 7B 法律模型
→ 所有数据本地处理，不出电脑

"审核这份合同的风险点"
→ 完全离线处理
→ 客户资料绝对安全
```

- **核心能力**：可选本地模型（不是强制的）
- **金句**："需要的时候，数据可以完全不出电脑"
