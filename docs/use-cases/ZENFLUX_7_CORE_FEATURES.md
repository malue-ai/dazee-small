# ZenFlux Agent 7大核心功能 Use Cases

本文档定义 ZenFlux Agent 的 7 个核心功能，并为每个功能提供 2-3 个典型 use case，包括用户 query 和期望的输出结果描述。

---

## 1. 智能内容生成（PPT/文档/报告）

### 功能描述
支持基于用户需求自动生成高质量的 PPT、Word 文档、PDF 报告，整合素材搜索、内容规划、专业渲染的完整流程。

### Use Case 1.1: 快速生成产品介绍 PPT

**用户 Query:**
```
帮我制作一个关于"智能客服系统"的产品PPT，包括产品优势、核心功能、客户案例，目标受众是企业管理层，风格要商务专业。
```

**期望输出:**
- 生成 8-12 页专业 PPT
- 包含封面、目录、产品概述、核心功能（3-4页）、客户案例、联系方式
- 商务风格模板，配有相关图表和配图
- 返回可下载的 PPTX 文件和预览链接
- 每页附带演讲者备注（可选）

### Use Case 1.2: 生成技术调研报告

**用户 Query:**
```
我需要一份关于"2024年AI大模型发展趋势"的调研报告，Word格式，至少3000字，包含市场数据、技术分析和未来展望。
```

**期望输出:**
- 生成 Word 文档（.docx）
- 自动搜索最新的行业报告和数据
- 包含结构化章节：摘要、市场概况、技术趋势分析、竞争格局、未来预测
- 配有图表、表格和数据引用
- 字数 3000-5000 字
- 附参考文献列表

### Use Case 1.3: 生成会议纪要

**用户 Query:**
```
根据我上传的会议录音转文本，帮我生成一份会议纪要，提取关键决策、行动项和责任人。
```

**期望输出:**
- 结构化的会议纪要文档（Word 或 PDF）
- 包含：会议基本信息、参会人员、讨论要点、关键决策、待办事项（含责任人和截止日期）
- 自动识别并高亮重要信息
- 生成待办事项清单（可导出为任务管理工具格式）

## 2. 深度数据分析与可视化

### 功能描述
支持上传多个数据文件（Excel/CSV），通过自然语言查询进行数据分析，自动生成图表、SQL 查询和分析报告。

### Use Case 2.1: 销售数据分析

**用户 Query:**
```
我上传了2024年全年的销售数据（sales_2024.xlsx），帮我分析一下：
1. 哪个季度销售额最高？
2. 哪个区域表现最好？
3. 生成月度销售趋势图。
```

**期望输出:**
- 返回结构化分析结果：
  - Q4 销售额最高（具体数字）
  - 华东区域销售额占比 35%，排名第一
- 生成 3 张图表：
  - 季度销售对比柱状图
  - 区域销售分布饼图
  - 月度销售趋势折线图
- 附 SQL 查询语句和数据摘要表格
- 支持导出为 Excel 报告

### Use Case 2.2: 多文件数据对比分析

**用户 Query:**
```
我有两个文件：2023年销售数据.xlsx 和 2024年销售数据.xlsx，帮我对比分析同比增长情况，找出增长最快的产品类别。
```

**期望输出:**
- 自动合并两个文件的数据
- 计算同比增长率（总体 +15%）
- 识别增长最快的产品类别（如"智能硬件"类增长 40%）
- 生成对比图表：
  - 年度总销售额对比
  - 产品类别增长率排行
  - Top 5 增长产品明细表
- 附数据洞察和建议（如"建议加大智能硬件类投入"）

### Use Case 2.3: 客户行为分析

**用户 Query:**
```
分析用户行为数据（user_behavior.csv），找出高价值客户特征和流失风险用户。
```

**期望输出:**
- 高价值客户画像（消费频次 > 10次，客单价 > 500元，占比 15%）
- 流失风险用户列表（90天未活跃，共 120 人）
- 生成可视化：
  - RFM 模型分析图
  - 客户价值分层金字塔
  - 流失风险热力图
- 提供运营建议（如"建议对流失风险用户发送优惠券"）

## 3. 智能信息检索与知识管理

### 功能描述
支持从互联网、企业知识库、个人文档中检索信息，提供语义搜索和精准摘要能力。

### Use Case 3.1: 个人知识库检索

**用户 Query:**
```
我记得之前上传过一份关于"项目管理方法论"的资料，帮我找一下里面关于敏捷开发的部分。
```

**期望输出:**
- 从用户的个人知识库（Ragie）中检索相关文档
- 返回匹配结果：
  - 文档名称：《项目管理方法论汇编.pdf》
  - 相关章节：第3章 敏捷开发方法（第 15-28 页）
  - 核心摘要：敏捷开发的四大价值观、Scrum 框架、Sprint 流程
- 支持继续追问（如"什么是 Sprint？"）

### Use Case 3.2: 行业资讯搜索

**用户 Query:**
```
搜索最近3个月关于"OpenAI GPT-5"的新闻报道和分析文章。
```

**期望输出:**
- 使用 Exa Search 进行语义搜索
- 返回 10 条高质量结果：
  - 新闻标题、来源、发布日期
  - 每条结果的核心摘要（100-200字）
  - 按时间倒序排列
- 自动过滤低质量内容
- 支持按类别筛选（news/papers/github）

### Use Case 3.3: 竞品调研

**用户 Query:**
```
帮我调研一下国内主要的AI客服产品，对比他们的功能、定价和客户评价。
```

**期望输出:**
- 自动搜索和抓取竞品信息（如百度、谷歌、产品官网）
- 生成竞品对比表格：
  - 产品名称、核心功能、价格区间、目标客户、优缺点
  - 至少覆盖 5 个主流产品
- 汇总分析：
  - 市场主流定价：5000-50000元/年
  - 共性功能：智能问答、工单管理、数据分析
  - 差异化功能对比
- 生成竞品分析报告（Word/PDF）

## 4. 零代码应用生成

### 功能描述
基于用户需求快速生成可交互的 Web 应用（Streamlit/Gradio/Vue/React），支持数据应用、工具应用、管理后台等场景。

### Use Case 4.1: 生成数据可视化应用

**用户 Query:**
```
帮我创建一个销售数据看板应用，用户可以上传Excel文件，自动生成销售趋势图和区域分布图，要求界面简洁易用。
```

**期望输出:**
- 生成 Streamlit 应用（Python）
- 功能模块：
  - 文件上传组件（支持 .xlsx/.csv）
  - 自动数据解析和清洗
  - 生成 3 类图表：趋势图、柱状图、饼图
  - 支持日期范围筛选
- 返回预览 URL（沙箱运行）
- 提供源代码下载（ZIP 包）
- 包含部署说明（如何部署到 Streamlit Cloud）

### Use Case 4.2: 生成 CRM 线索管理系统

**用户 Query:**
```
我需要一个简单的销售线索管理系统，包括线索录入、跟进记录、状态变更，数据存储在本地JSON文件就行。
```

**期望输出:**
- 生成 React + Ant Design 应用
- 功能模块：
  - 线索列表（表格展示）
  - 新增/编辑线索（表单弹窗）
  - 跟进记录（时间线展示）
  - 状态流转（待跟进 → 跟进中 → 已成交/已流失）
  - 本地 JSON 文件存储
- 返回预览 URL 和源代码
- 支持后续迭代（如"加一个搜索功能"）

### Use Case 4.3: 生成文档转换工具

**用户 Query:**
```
创建一个 PDF 转 Word 的 Web 工具，用户上传 PDF 文件，自动转换后下载 Word 文档。
```

**期望输出:**
- 生成 Flask/Gradio 应用
- 功能模块：
  - 文件上传（支持拖拽）
  - PDF 解析和转换（调用 pdf2docx 库）
  - 进度条显示
  - 转换完成后自动下载
- 返回预览 URL
- 处理异常情况（如加密 PDF、扫描件）

## 5. 任务规划与执行

### 功能描述
自动将复杂任务拆解为可执行的步骤，支持进度跟踪、动态调整和自适应重规划。

### Use Case 5.1: 市场调研任务规划

**用户 Query:**
```
我要做一个"智能家居市场"的深度调研，帮我规划一下需要做哪些事情。
```

**期望输出:**
- 生成任务计划（Plan）：
  1. 搜索行业报告和市场数据（pending）
  2. 分析主要竞品（3-5家）（pending）
  3. 总结市场规模和增长趋势（pending）
  4. 识别目标客户群体（pending）
  5. 生成调研报告（Word）（pending）
- 显示预计耗时：30-45 分钟
- 开始执行第一步（自动搜索）
- 实时更新步骤状态（in_progress → completed）
- 支持用户干预（如"跳过竞品分析"）

### Use Case 5.2: 复杂报告生成任务

**用户 Query:**
```
帮我生成一份Q4季度运营报告，包括销售数据分析、用户增长情况、运营活动回顾，最后输出PPT。
```

**期望输出:**
- 生成多步骤计划：
  1. 读取 Q4 销售数据文件（completed）
  2. 分析销售数据（completed）
  3. 分析用户增长数据（in_progress）
  4. 梳理运营活动清单（pending）
  5. 规划 PPT 结构（pending）
  6. 生成 PPT（pending）
- 每个步骤完成后显示结果摘要
- 最终交付：20 页 PPT + 数据分析 Excel

### Use Case 5.3: 自适应重规划

**用户 Query:**
```
（任务执行中）这个方向不对，我其实想要的是对比分析，不是趋势分析。
```

**期望输出:**
- Agent 识别到需求变更
- 触发 replan（重新规划）
- 保留已完成的步骤（数据读取、清洗）
- 重新生成后续步骤：
  3. 进行同比/环比对比分析（new）
  4. 生成对比图表（new）
  5. 输出对比报告（new）
- 继续执行新计划

## 6. API 集成与自动化

### 功能描述
支持调用外部 API（CRM、ERP、第三方服务），实现跨系统数据获取和自动化流程。

### Use Case 6.1: CRM 客户信息查询

**用户 Query:**
```
从 CRM 系统中查询"张伟"这个客户的详细信息，包括联系方式、历史订单和跟进记录。
```

**期望输出:**
- 自动调用 CRM API（需预配置）
- 返回结构化信息：
  - 基本信息：姓名、公司、职位、手机、邮箱
  - 历史订单：共 5 笔，总金额 12 万元
  - 最近跟进：2024-01-15，状态"意向强烈"
  - 负责销售：李明
- 支持后续操作（如"帮我发一封跟进邮件"）

### Use Case 6.2: 自动化报销流程

**用户 Query:**
```
帮我提交一笔差旅报销，金额 3500 元，包括高铁票 800 元和住宿费 2700 元，发票已上传。
```

**期望输出:**
- 调用 ERP 报销 API
- 自动填充报销单：
  - 报销类型：差旅费
  - 明细：高铁票 + 住宿费
  - 发票附件：自动上传
  - 审批流：提交给直属上级
- 返回报销单号：BX20240115001
- 提示："报销单已提交，预计 3 天内审批完成"

### Use Case 6.3: 跨系统数据同步

**用户 Query:**
```
把本月新增的客户信息从 CRM 导出，然后批量导入到邮件营销系统。
```

**期望输出:**
- 步骤 1：从 CRM API 获取本月新增客户（共 120 条）
- 步骤 2：数据清洗和格式转换（统一字段格式）
- 步骤 3：调用邮件营销系统 API 批量导入
- 步骤 4：返回结果：
  - 成功导入：115 条
  - 失败：5 条（原因：邮箱格式错误）
  - 生成失败记录 Excel 供人工处理

## 7. 人机协作与确认机制

### 功能描述
在执行关键操作前主动请求用户确认，支持多选项决策和文本输入，实现 Human-in-the-Loop（HITL）。

### Use Case 7.1: 删除操作确认

**用户 Query:**
```
删除所有 2022 年之前的客户数据。
```

**期望输出:**
- Agent 暂停执行，弹出确认对话框：
  - 问题："即将删除 2022 年之前的客户数据（共 350 条），此操作不可恢复，是否确认？"
  - 选项：["确认删除", "取消", "先导出备份再删除"]
- 用户选择："先导出备份再删除"
- Agent 继续执行：
  1. 导出数据为 Excel（backup_2022_before.xlsx）
  2. 执行删除操作
  3. 返回："已删除 350 条数据，备份文件已保存"

### Use Case 7.2: 方案选择确认

**用户 Query:**
```
帮我生成一份产品介绍 PPT。
```

**期望输出:**
- Agent 识别到有多种风格选项，弹出确认对话框：
  - 问题："请选择 PPT 风格"
  - 选项：["商务专业", "学术严谨", "创意时尚", "极简主义"]
- 用户选择："商务专业"
- Agent 继续执行 PPT 生成

### Use Case 7.3: 参数补充确认

**用户 Query:**
```
帮我发一封邮件给张经理。
```

**期望输出:**
- Agent 检测到缺少必要信息，弹出确认对话框：
  - 问题："请补充以下信息"
  - 输入框：
    - 收件人邮箱：_______
    - 邮件主题：_______
    - 邮件正文：_______（可选，Agent 可帮忙撰写）
- 用户填写后 Agent 继续执行发送邮件

## 功能与工具能力映射

| 核心功能 | 主要使用的工具/能力 | 备注 |
|---------|-------------------|------|
| 1. 智能内容生成 | `ppt_generator`, `docx`, `pdf`, `web_search` | 整合搜索和渲染 |
| 2. 深度数据分析 | `wenshu_analytics`, `xlsx`, `data_visualization` | 支持自然语言查询 |
| 3. 智能信息检索 | `knowledge_search`, `exa_search`, `web_search` | 语义搜索 + 摘要 |
| 4. 零代码应用生成 | `sandbox_create_project`, `sandbox_run_project` | Streamlit/React/Vue |
| 5. 任务规划与执行 | `plan_todo`, `Extended Thinking` | 自适应 replan |
| 6. API 集成与自动化 | `api_calling`, CRM/ERP 集成能力 | 通用 API 调用 |
| 7. 人机协作确认 | `request_human_confirmation` | HITL 机制 |

## 场景组合示例

### 综合场景：季度总结报告生成

**用户 Query:**
```
帮我生成 Q4 季度总结报告，包括销售数据分析、运营活动回顾、竞品分析，最后输出成 PPT。
```

**涉及的功能:**
1. **任务规划**（功能 5）：拆解为 6 个步骤
2. **数据分析**（功能 2）：分析 Q4 销售数据
3. **信息检索**（功能 3）：搜索竞品动态
4. **人机协作**（功能 7）：确认 PPT 风格和大纲
5. **内容生成**（功能 1）：生成最终 PPT

**期望输出:**
- 完整的任务执行流程（30-45 分钟）
- 中间结果：数据分析图表、竞品摘要、运营活动清单
- 最终交付：25 页专业 PPT + 数据分析 Excel

## 总结

以上 7 大功能覆盖了 ZenFlux Agent 的核心应用场景，每个功能通过 2-3 个典型 use case 展示了：
- **用户输入**：自然语言 query
- **Agent 行为**：调用哪些工具、执行哪些步骤
- **期望输出**：具体的交付物和结果描述

这些 use cases 可以作为：
- **产品设计**的参考
- **测试用例**的基础
- **用户文档**的素材
- **演示 Demo**的脚本
