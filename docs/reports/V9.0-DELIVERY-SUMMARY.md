## 🎯 核心改进点

### 1. 上下文过滤策略

**原有问题**：
- 简单截取最近 30 条消息
- 包含大量工具调用结果（5-10K tokens）
- 智能体冗长回复污染上下文

**优化方案**：
- 只保留最近 5 条 user 消息
- 最后 1 条 assistant 截断为 100 字符
- 直接丢弃 tool_use/tool_result

**实测效果**：
```
原始: 21 条消息 (~20K tokens)
过滤: 6 条消息 (~5.6K tokens)
过滤率: 71.4%
```

### 2. 意图状态转换

**原有问题**：
- 追问时重新分析全量上下文
- 未复用 plan_cache
- 未显式处理状态转换

**优化方案**：
```python
if is_follow_up:
    # 追问：保留状态
    self._plan_cache = self._plan_cache  # 不重置
    new_intent.task_type = previous_intent.task_type
else:
    # 新意图：重置状态
    self._plan_cache = {"plan": None, "todo": None, "tool_calls": []}
```

**实测效果**：
- ✅ 日志显示：`🔄 追问场景优化: 继承上轮 task_type`
- ✅ 日志显示：`🆕 新意图: 重置 plan_cache`
