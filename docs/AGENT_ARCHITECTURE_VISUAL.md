## 4️⃣ 关键设计特点

### ✅ 异步架构
```python
# 所有关键方法都是 async def
async def chat()                    # Agent 入口
async def create_message_async()    # LLM 调用
async def execute()                 # 工具执行
async def emit_*()                  # 事件发送
```

### ✅ 流式优化
```python
# create_message_stream() 是真正的异步生成器
async for llm_response in stream_generator:
    yield event  # 实时输出
    await asyncio.sleep(0)  # 🔑 让出控制权，防止阻塞
```

### ✅ 事件驱动
```python
# 统一的事件格式
{
    "event_uuid": "...",     # UUID (不是全局 ID)
    "seq": 1,                # Session 内序号
    "type": "content_delta", # 事件类型
    "session_id": "...",     # 会话 ID
    "conversation_id": "...",# 对话 ID
    "timestamp": "...",      # ISO 时间戳
    "data": {...}            # 事件数据
}
```

### ✅ 双 LLM 架构
```
Intent Recognition (Haiku 4.5)
- 快速分类 (~0.5s)
- 便宜 (1/10 价格)
- 无 Extended Thinking

         ↓

Execution (Sonnet 4.5)
- Extended Thinking
- 工具调用
- 高质量输出
```

### ✅ 数据持久化
```
Agent 执行时:
- user message → DB (立即保存)
- assistant message → DB (complete 事件时保存)
- thinking content → message.metadata

前端重载:
- 从 DB 加载历史消息
- 继续对话 (带上下文)
```

## 5️⃣ 消息格式转换链

```
前端发送:
{message: "你好"}  (字符串)

         ↓ Router 标准化

Claude API 格式:
[{"type": "text", "text": "你好"}]  (数组)

         ↓ Agent 执行

历史消息格式 (传给 LLM):
[
  {"role": "user", "content": "你好"},
  {"role": "assistant", "content": "你好！有什么可以帮你的吗？"}
]

         ↓ 保存到数据库

数据库格式 (JSON 字符串):
content: '[{"type": "text", "text": "你好！有什么可以帮你的吗？"}]'
status: '{"index": 0, "action": "completed", "description": "回复完成"}'

         ↓ 返回给前端

前端接收 (已解析):
{
  content: [{"type": "text", "text": "你好！有什么可以帮你的吗？"}],
  status: {"index": 0, "action": "completed", "description": "回复完成"}
}
```
