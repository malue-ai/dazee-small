# Skills Usage Guidelines

## For Pre-built Skills (pptx, xlsx, docx, pdf)

These Skills work automatically without needing to read SKILL.md:
- They have standard workflows built-in
- Just use them naturally in your tasks
- Example: 'Create a presentation about AI' will automatically use the pptx Skill

## For Custom Skills (planning-task, slidespeak-generator, etc.)

These require loading their SKILL.md for detailed instructions:

1. **Identify**: Check the Skills list above
2. **Load**: Use `bash: cat skills/<skill-name>/SKILL.md`
3. **Follow**: Execute the workflow in SKILL.md
4. **Access resources**: Load scripts/ and resources/ as directed

**Example workflow**:
```bash
# 1. Load Skill instructions
cat skills/slidespeak-generator/SKILL.md

# 2. Load API schema (if needed)
cat skills/slidespeak-generator/resources/api_schema.json

# 3. Execute helper script
python -c "
exec(open('skills/slidespeak-generator/scripts/config_builder.py').read())
config = build_slidespeak_config(topic='AI Product', pages=5)
print(config)
"
```

## Runtime Environments vs Skills

**Important**: Bash, Python, Nodejs are NOT Skills.
- They are runtime environments in the VM
- Access them via code_execution tool
- No discovery or loading needed

## Priority and Routing Rules

When multiple Skills can accomplish the same task:

1. **Check explicit mentions**: If user mentions a specific Skill/Tool name, use it
2. **Apply keyword matching**: Match user keywords with Skill's 'preferred_for' list
3. **Follow priority order**: High (🥇) > Medium (🥈) > Low (🥉)
4. **Consider context**: Continue with same Skill for iterations/refinements

**Example: PPT Generation**
```
User: "创建产品介绍PPT"
→ Keywords: 产品介绍 (implies professional/business)
→ Available: slidespeak-generator (High, preferred_for: 'professional PPT')
→ Decision: Use slidespeak-generator (priority + keyword match)

User: "快速生成PPT草稿"
→ Keywords: 快速, 草稿 (implies quick/simple)
→ Available: pptx (Pre-built, faster and simpler)
→ Decision: Use pptx (better fit for quick draft)
```

**Decision Protocol**:
Use Extended Thinking to evaluate options when multiple Skills are available.

**Remember**: Skills provide expertise, Tools provide execution. Use them together.
